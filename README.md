# 业务知识库模板

这是一份去业务化的知识库骨架，用于帮助团队快速建立自己的业务知识库。模板保留了目录结构、智能体工作流、文档规范、lint 脚本和少量虚构示例；不包含任何真实业务规则、组织名称、人员信息、系统专有名称或流程细节。

## 使用方式

1. 复制本目录并改名为你的知识库名称。
2. 打开 `AGENTS.md`，把所有 `<待填写>` 和 `示例...` 替换为你的真实业务边界、系统清单、角色清单和维护规则。
3. 如果是第一次创建知识库，先执行 `BOOTSTRAP_ONCE.md`；初建完成后删除或归档该文件，后续维护不要再引用。
4. 把原始材料追加放入 `RAW_SOURCES/`，不要修改或删除原始材料。
5. 在 `WIKI/` 中维护结构化知识，保持 `index.md`、各层 `_index.md`、`log.md` 同步。
6. 每次维护后运行：

```bash
python3 skills/lint/scripts/lint_wiki.py
```

## 初始化专用文件

`BOOTSTRAP_ONCE.md` 是一次性建库清单，只用于从空模板创建知识库的第一轮初始化。完成业务边界、菜单骨架、初始 index 和 log 后，应删除该文件，或移动到 `文档备份/` 并在 `WIKI/log.md` 记录“首次建库完成”。

后续维护知识库时，不要再引用 `BOOTSTRAP_ONCE.md`，只按 `AGENTS.md`、目标 `_index.md` 和对应 skill 执行。

## 信息添加模式

真实知识库建议按以下优先级添加信息：

1. **页面实采优先**：以当前系统页面可见的菜单、字段、按钮、弹窗、提示和线上行为为第一事实来源。
2. **源码分析补充（后续可选）**：当需要解释隐藏校验、接口链路、状态机、算法规则或页面背后逻辑，且能获取源码或接口资料时，再读取本地代码，记录到 `WIKI/源码映射/`，并把业务结论回写到功能模块、概念或流程页。
3. **RAW 提取补充**：从操作手册、QA 集、设计文档、表格、历史截图和专家补充中提炼知识。RAW 能补全背景和例外，但如果和页面实采冲突，应先标记待确认。

三种模式分别对应：

| 模式 | 典型输入 | 主要产出 | Skill |
|------|----------|----------|-------|
| 页面实采 | 用户已登录的业务系统页面、菜单路径、截图上下文 | `RAW_SOURCES/截图/`、`WIKI/功能模块/`、操作流程入口 | `record-feature-to-wiki` |
| 源码分析（后续可选） | 本地源码、接口定义、算法服务、枚举类 | `WIKI/源码映射/`、规则补充、页面到代码证据链 | `extract-source-to-wiki` |
| RAW 提取 | 操作手册、QA 集、设计文档、表格、专家补充 | 概念、流程、规则、字段说明和来源索引 | `query` + `lint` |

## 模板中保留的示例

模板只保留了一个虚构概念、一个虚构页面、一个虚构流程和一个可选源码映射入口，用来演示 frontmatter、索引、来源引用和文档颗粒度。源码映射不是首次建库必需项，它们都不是业务建议，也不应被当成真实规则。

## 脱敏原则

- 不写真实公司、部门、客户、人员、系统域名、账号、订单号、合同号等信息。
- 截图如包含敏感数据，应先脱敏后再放入可分享模板。
- 模板中的 `RAW_SOURCES/` 只放 README 或虚构示例，不放真实原始材料。
- 对外分享前清理 `.idea/`、`.DS_Store`、临时文件和真实下载文件。
- 对外分享前可执行：

```bash
rg -n "公司名|系统名|人员名|手机号|邮箱|合同号|订单号" .
python3 skills/lint/scripts/lint_wiki.py
python3 skills/lint/scripts/lint_wiki.py --strict-init
```
