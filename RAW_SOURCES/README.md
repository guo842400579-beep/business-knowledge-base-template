# RAW_SOURCES

这里存放不可变原始来源。规则：只追加，不修改，不删除。

页面实采时，不要把所有候选截图、临时 DOM 和失败尝试直接放入本目录。先在 `.work/page-capture/` 探索和筛选，只把能独立支撑最终结论的最小充分证据提升到 RAW。

| 目录 | 放什么 | 脱敏要求 |
|------|--------|----------|
| `设计文档/` | 需求文档、字段模板、导入样例、流程说明 | 删除真实客户、账号、单号、金额等敏感值 |
| `专家知识/` | 专家口头或文字补充原文 | 标注日期和来源角色，不写真实姓名 |
| `截图/` | 页面、弹窗、列表、配置、流程截图 | 提升前逐图检查，在 `.work` 生成脱敏副本 |
| `用户反馈/` | 用户对 AI 回答或文档的纠正原文 | 保留原意，去除个人身份信息 |

以下内容只留在 `.work/`，不进入 RAW：导航过程、重复截图、定位控件用 DOM、按钮清单、完成凭证、工具报错和恢复点。

提升永久证据时使用唯一文件名；目标已存在必须改用新时间戳或序号，不得覆盖。候选来源、脱敏结果、目标路径和 SHA-256 记录在对应 `.work` 批次的证据清单中。
