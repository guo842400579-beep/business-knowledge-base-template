# 待确认项索引

本目录只登记会影响业务结论的决策问题。采集队列、完成进度、控件盘点、临时截图、源码扫描进度和恢复点属于 `.work/`，不得写入本目录。

## 状态模型

| 主状态 | 含义 | 关闭条件 |
|--------|------|----------|
| 待业务确认 | 页面、RAW、源码或既有 WIKI 存在需要业务人员裁决的口径差异 | 业务口径已确认并留有来源 |
| 待页面实采 | 需要特定角色、数据、状态或权限才能验证页面行为 | 页面证据已采集，或明确批准排除 |
| 待运行验证 | 静态证据不足，需要测试环境、日志或实际执行结果 | 运行证据已获得并完成结论回写 |
| 已确认待回写 | 决策已完成，但相关业务页、索引或流程尚未全部同步 | 所有受影响文档、索引和日志已更新并通过 lint |
| 已关闭 | 结论已回写，或问题已按明确理由批准排除 | 无剩余回写动作 |

## 登记字段

事项较多时一项一页，使用 `type: decision`，引用触发该决策的具体 RAW 文件，并至少记录：

| 字段 | 要求 |
|------|------|
| 编号与标题 | 使用稳定编号，标题直接描述待决策问题 |
| 主状态 | 只能使用上表中的一个值 |
| 业务背景 | 说明为什么该问题会影响当前结论 |
| 正式业务口径 | 已确认则写明；未确认不得猜测 |
| 当前页面行为 | 仅记录可见或已实采行为 |
| 源码实现 | 记录仓库版本边界，不把实现自动提升为业务规则 |
| 实现差异 | 分开描述正式规则、页面行为和实现缺口 |
| 验证方式 | 需要谁确认、补采什么页面或执行什么验证 |
| 影响文档 | 列出确认后必须回写的功能、概念、流程或源码映射页 |
| 剩余问题 | 说明尚未关闭的边界；无则写“无” |

单项 frontmatter：

```yaml
---
title: 待确认事项标题
type: decision
source: |
  RAW_SOURCES/用户反馈/用户原话.md
date: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [待确认]
related: [受影响文档]
---
```

## 虚构示例

| 编号 | 事项 | 主状态 | 验证方式 | 影响范围 |
|------|------|--------|----------|----------|
| DEMO-001 | 页面值 `Draft`、`Pending` 如何对应中文业务状态 | 待业务确认 | 由业务负责人确认中英文状态映射，并补采状态下拉与提交入口 | 示例概念、示例页面、示例流程 |
