# 源码映射索引

源码映射用于记录页面动作、接口、类、方法、算法入口和代码版本之间的证据链。它是追溯层，不是业务知识的最终阅读层。

## 证据边界

1. 功能页面直接镜像为 `WIKI/源码映射/<系统>/<菜单>/<页面>.md`，不额外增加“功能模块”一层。
2. 概念和流程分别落在 `WIKI/源码映射/概念/`、`WIKI/源码映射/操作流程/`，文件名与业务正文一致。
3. 业务结论必须回写到对应功能模块、概念或操作流程页面。
4. 本目录只保留代码证据、路径、版本、接口链路和待确认差异。
5. 如果源码与页面实采、操作手册或用户反馈冲突，先记录待确认问题，不直接覆盖页面事实。
6. 映射正文使用 `type: source-map`，并引用目标业务对象已有的具体 RAW 证据；源码仓库、分支、完整 commit 和工作区指纹写在正文。
7. 没有业务对象或永久证据支撑的代码候选留在 `.work/source-analysis/`，不得把正文伪装成 `type: index` 绕过 source 校验。
