# 首次建库清单（一次性）

> 本文件只用于从空模板创建知识库的第一轮初始化。
>
> 完成后必须删除，或移动到 `文档备份/BOOTSTRAP_ONCE_已完成_YYYY-MM-DD.md`。后续维护只读 `AGENTS.md`、相关 index 和对应 Skill。

## 完成标准

1. `AGENTS.md` 已写清业务边界、明确非目标、系统、角色、事实时效和敏感信息规则。
2. `.gitignore` 已忽略 `.work/`，临时采集和源码分析不会进入版本库。
3. `WIKI/功能模块/` 已按真实菜单建立首版骨架。
4. `RAW_SOURCES/截图/` 已按同一套业务对象建立永久证据路径。
5. `WIKI/index.md`、概念 index 和相关 `_index.md` 已能导航到初始结构。
6. `WIKI/待确认项/` 已保留入口，但没有把采集进度写成待确认业务事实。
7. `WIKI/log.md` 已用不超过两条 bullet 记录首次建库。
8. 默认 lint 通过，`--audit` 已单独查看。

## 1. 基础问题

| 问题 | 用途 |
|------|------|
| 知识库负责回答哪个业务域？ | 定义权威范围 |
| 哪些相邻或外部业务明确不维护？ | 防止边界外内容反复成为缺口 |
| 涉及哪些系统和端？ | 决定功能模块第一层目录 |
| 每个系统的主要角色是谁？ | 决定页面说明和权限边界 |
| 哪些信息不能写入知识库或分享版？ | 建立脱敏和敏感词规则 |
| 页面、RAW、源码和人工确认的版本边界是什么？ | 防止把旧证据当当前事实 |
| 是否能访问系统页面？ | 决定页面实采范围 |
| 是否有手册、QA、历史截图、表格或源码？ | 决定 RAW 与源码维护路径 |

## 2. 首轮目录

| 层级 | 首轮目标 |
|------|----------|
| `.work/page-capture/` | 建立临时采集约定；不提交目录内容 |
| `RAW_SOURCES/截图/` | 与真实菜单结构对齐，只接收筛选后的证据 |
| `WIKI/功能模块/` | 建立系统、菜单、页面骨架 |
| `WIKI/概念/` | 维护正式术语、角色和核心对象 |
| `WIKI/操作流程/` | 只创建已经有来源支撑的核心流程 |
| `WIKI/源码映射/` | 首次建库可只保留入口，后续有源码再启用 |
| `WIKI/待确认项/` | 仅登记影响业务结论的决策问题 |

首轮先形成可导航结构，不急于填满规则。Index 只记录稳定入口和摘要，不写完成打勾、采集日期或恢复点。

## 3. 页面采集起步

1. 确认系统、环境、角色和截图脱敏规则。
2. 从首页或侧边栏开始，只做安全、可逆的探索。
3. 候选截图、临时 DOM 和菜单队列先放 `.work/page-capture/`。
4. 筛选后再把最小充分证据提升到 `RAW_SOURCES/截图/`。
5. 页面字段、按钮和规则必须按单页面闭环写入 WIKI。
6. 最终保存、提交、确定、删除确认等写入动作必须另行申请权限。

## 4. 初始验证

运行检查前先完成占位清理：

1. 替换 `AGENTS.md` 中的业务边界、系统和角色占位。
2. 删除虚构的示例概念、页面、流程和 `DEMO-001`，或将它们完整替换为真实内容。
3. 删除 `RAW_SOURCES/截图/示例系统/`，并让所有 WIKI `source` 改指真实、已脱敏的永久证据。
4. 重建各级 Index，清除“示例系统 / 示例页面 / 示例流程”等虚构路径。
5. 把 `WIKI/log.md` 重置为真实知识库的首次建库记录；不要保留模板演进历史。

```bash
python3 skills/lint/scripts/lint_wiki.py
python3 skills/lint/scripts/lint_wiki.py --audit
git diff --check
```

完成占位替换后再运行：

```bash
python3 skills/lint/scripts/lint_wiki.py --strict-init
```

## 5. 失效处理

删除或归档本文件后，在 `WIKI/log.md` 追加：

```markdown
## [YYYY-MM-DD] init | 首次建库完成
- 已完成业务边界、菜单骨架、初始索引和证据路径。
- 已建立首批可追溯业务内容，并清除模板占位。
```
