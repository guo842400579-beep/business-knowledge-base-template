# 首次建库清单（一次性）

> 本文件只用于从空模板创建知识库的第一轮初始化。
>
> 完成初步创建后，必须删除本文件，或移动到 `文档备份/BOOTSTRAP_ONCE_已完成_YYYY-MM-DD.md`。后续维护知识库时，不得再引用本清单。

## 完成标准

当以下事项完成后，本文件即失效：

1. `AGENTS.md` 已替换业务边界、系统清单、角色清单和敏感信息规则。
2. `WIKI/功能模块/` 已按真实系统菜单建立第一版目录骨架。
3. `RAW_SOURCES/截图/` 已按同一套菜单结构建立截图归档路径。
4. `WIKI/index.md` 和相关 `_index.md` 已能导航到初始页面骨架。
5. `WIKI/log.md` 已记录首次建库完成。
6. `python3 skills/lint/scripts/lint_wiki.py` 通过。

## 1. 基础问题

| 问题 | 用途 |
|------|------|
| 这个知识库覆盖哪个业务域？ | 写入 `AGENTS.md` 的业务边界 |
| 涉及哪些业务系统？ | 决定 `WIKI/功能模块/` 第一层目录 |
| 每个系统的主要角色是谁？ | 决定页面说明和权限边界 |
| 哪些信息不能写进可分享知识库？ | 建立敏感词和脱敏规则 |
| 是否能访问业务系统页面？ | 决定是否启动页面实采 |
| 是否有操作手册、QA 集、历史文档或截图？ | 决定 RAW 提取范围 |

## 2. 菜单结构优先

首次建库优先让页面实采证据和功能模块文档跟真实系统菜单保持一致：

| 结构 | 推荐对齐方式 |
|------|--------------|
| `RAW_SOURCES/截图/` | `系统/一级菜单/二级菜单/页面/截图文件` |
| `WIKI/功能模块/` | `系统/一级菜单/二级菜单/页面.md` |

建库第一轮不急着写完整规则，优先形成菜单树、页面入口、页面标题、URL 脱敏规则和截图归档路径。等架构稳定后，再逐页补字段、按钮、流程和规则。

## 3. 浏览器接管建议

如果用户允许智能体查看业务系统页面，建议按以下顺序执行：

1. 确认系统名称、环境标识、登录角色和是否允许截图。
2. 从系统首页或侧边栏开始，只做只读操作。
3. 逐级展开菜单，记录菜单名称、页面标题、URL 脱敏形式和页面层级。
4. 按菜单层级创建 `WIKI/功能模块/` 和 `RAW_SOURCES/截图/` 目录。
5. 对每个页面保存首屏截图，后续再按页面实采模式补全字段、按钮、弹窗、宽表和流程。
6. 不点击提交、保存、删除确认等落库动作；如必须验证，先向用户申请权限。

## 4. 失效处理

初始化完成后执行其一：

```bash
rm BOOTSTRAP_ONCE.md
```

或：

```bash
mv BOOTSTRAP_ONCE.md 文档备份/BOOTSTRAP_ONCE_已完成_YYYY-MM-DD.md
```

然后在 `WIKI/log.md` 追加：

```markdown
## [YYYY-MM-DD] init | 首次建库完成
- 已完成业务边界、菜单骨架、初始索引和 lint。
- `BOOTSTRAP_ONCE.md` 已删除或归档，后续维护不再引用首次建库清单。
```
