# 维护样例摘要

本文件只保留去业务化样例，用于说明实采页面时的工作方式。

## 示例：宽表页面采集

关键做法：

1. 先确认页面标题、菜单路径和 URL。
2. 首屏截图只能证明首屏可见内容，不能代表完整列表。
3. 横向滚动到最右，合并完整列头。
4. 固定列要按视觉顺序记录，例如“选择框 → 操作按钮 → 数据列”。
5. 批量按钮要验证未勾选和已勾选两种状态。
6. 导出或下载模板要保存到 `RAW_SOURCES/设计文档/`，读取字段后写入 WIKI。
7. 更新页面文档、最近 `_index.md`、`WIKI/index.md` 和 `WIKI/log.md`。

## 示例：用户纠正处理

关键做法：

1. 用户纠正先原文追加到 `RAW_SOURCES/用户反馈/`。
2. 判断纠正影响哪些概念、页面和流程。
3. 更新 WIKI 时说明已按反馈修正。
4. 如果纠正会影响多个文档，建立任务清单逐个闭环。
5. 维护后运行 lint。
