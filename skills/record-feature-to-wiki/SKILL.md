# 记录系统功能到 WIKI

把当前页面事实转化为可追溯、可查询、可维护的知识。安全控件要实际探索，最终写入动作要止步并申请权限；候选证据先留 `.work`，筛选后才进入不可变 RAW。

## 前置阅读

依次读取：

1. `AGENTS.md`；
2. `WIKI/index.md`；
3. `WIKI/概念/概念_index.md`；
4. 目标目录 `_index.md`；
5. 即将更新的当前 WIKI 文件；
6. 多菜单、复杂页面或发布前读取 [页面实采发布与知识纯化](references/publishing-and-purity.md)。

## 初始化启动门

真实知识库开始实采前必须满足：

```bash
python3 skills/lint/scripts/lint_wiki.py --strict-init
```

- `BOOTSTRAP_ONCE.md` 已删除或归档；
- `AGENTS.md` 已写明真实业务边界、系统、角色和敏感信息规则；
- 示例 WIKI、示例截图路径和虚构待确认项已替换或删除；
- `--strict-init` 返回 0。

未通过时只执行初始化，不采集或发布真实业务页面。维护模板本身时不适用此门。

## 四层分离

| 层级 | 内容 | 规则 |
|------|------|------|
| `.work/page-capture/` | 队列、按钮清单、候选截图、临时 DOM、失败尝试、恢复点 | Git 忽略；中断时保留 |
| `RAW_SOURCES/` | 能独立支撑结论的最小充分证据 | 只增不删 |
| `WIKI/` | 稳定字段、规则、状态、流程、按钮行为和必要信息边界 | 默认业务查询 |
| `WIKI/log.md` | 变更范围和关键结论 | 每次最多两条 bullet |

不得把进度、按钮清单、完成凭证、临时 DOM、截图归档动作或工具报错写入 WIKI、Index、log 或 RAW。

## 单信息单元闭环

记录功能信息时，每次只处理一个页面、Tab、弹窗、侧拉框、配置项或按钮组：

```text
打开并确认
→ .work 候选采集
→ 安全控件穷尽
→ 筛选并提升 RAW
→ 写 WIKI
→ 更新 Index / 极简 log
→ lint / 纯化 / diff
→ 进入下一个
```

允许例外：首次只梳理菜单树时，可以快速打开多个入口建立骨架。一旦记录字段、按钮、规则、弹窗、当前值或流程，立即切回单单元闭环。

容量阈值：

- 简单只读列表：同一对话最多 3–4 个；
- 有 Tab、弹窗或行操作：最多 2 个；
- 复杂联动、状态流转或多步骤表单：1 个。

发生上下文压缩、连续浏览器异常、需要寻找多状态样本或出现延迟写 WIKI 倾向时，完成当前单元后停止扩展并留下恢复点。

## 临时工作目录

创建：

```text
.work/page-capture/<YYYY-MM-DD>-<scope>/
├── queue.md
├── controls.md
├── temporary-screenshots/
└── temporary-dom/
```

- `queue.md` 记录信息单元、状态和下一步。
- `controls.md` 记录控件、已安全探索层级、证据候选和止步原因。
- 完成 / 受阻 / 未完成只留 `.work`，不写 Index。
- 中断恢复先读 `.work`、已归档 RAW 和已发布 WIKI，不凭记忆补写。

## 页面实采

### 1. 确认页面

开始采集前确认当前已登录目标页面，并核对：

| 信息 | 要求 |
|------|------|
| 页面标题 | 按可见正式文字 |
| 菜单路径 | 按真实导航层级 |
| URL | WIKI 只留脱敏、稳定形式 |
| 环境或组织 | 如敏感，仅在对话和 `.work` 临时确认 |
| 角色或权限 | 记录角色类别，不写真实姓名、账号或敏感权限 |
| 数据范围 | 当前账号允许查看的组织、对象和时间范围 |
| 采集授权 | 允许截图、保存临时 DOM、下载模板或导出文件的范围 |
| 脱敏规则 | 人员、账号、客户、金额、单号、手机号、邮箱等处理方式 |

### 2. 穷尽页面结构

检查：

- Tab、锚点、步骤条和子页面；
- 筛选、更多条件、默认值和 placeholder；
- 顶部按钮、图标、文字入口、更多操作、批量和行操作；
- 列头、固定列、横向滚动后的所有列；
- 分页、排序、空状态、有数据状态和异常状态；
- 弹窗、侧拉框、二次确认、toast 和错误提示；
- 下拉联动、日期选择、文件选择器、导入、导出和下载模板；
- 状态依赖、权限提示和风险文案。

首屏不能代表完整页面。按钮名称也不能代表真实 UI 形态，安全入口必须实际打开。

### 3. 操作安全

可以直接执行的通常包括：

- 查询、展开、收起、切换 Tab；
- 查看下拉；
- 打开普通弹窗或侧拉框但不提交；
- 取消、关闭、返回。

必须止步并申请权限：

- 最终保存、提交、确定、确认、删除确认、批量生效；
- 二次确认中的肯定、继续或执行按钮；
- 已知首击即写入的动作。

“撤回、作废、启停、重试、删除”等状态入口可能首击写入。只有确认首击仅打开二次确认层时，才打开第一层并记录文案；风险不明时停在点击前。

以下入口不能按“查看”默认处理：

- 文件选择器、上传和导入：选中文件后可能立即上传；
- 发送、同步、生成、测试连接、标记已读、重算和外部跳转：首击可能产生状态或外部影响；
- 导出和下载：虽然通常不改系统数据，但会在本机落盘，并可能带出真实明细；
- 模板下载：仅在确认来源和落盘位置后执行。

导出或下载确有必要时，先落到本批 `.work`，检查内容和敏感信息后再决定是否提升为 RAW。风险不明时停在点击前。

即使用户允许写入，也优先停在二次确认层，避免无意义改动。

### 4. 筛选证据

候选文件先放 `.work`。只提升：

- 页面主体和完整结构；
- 宽表右侧、隐藏区域或动态联动状态；
- 有业务意义的弹窗、侧拉框、确认文案和风险提示；
- 原始模板或能证明关键差异的必要 DOM；
- 用户原话和专家原文。

提升前必须逐个执行：

1. 目检图片、PDF、Office、CSV 和下载文件，检查人员、账号、客户、金额、单号、手机号、邮箱、组织和环境标识。
2. 在 `.work` 生成脱敏副本；原始候选仍留临时层，不把未脱敏版本提升到 RAW。
3. 使用 `YYYY-MM-DD_HHMMSS_<状态>_<序号>.<扩展名>` 等唯一文件名。
4. 目标路径已存在时拒绝覆盖，改用新序号或新时间戳。
5. 在 `.work/evidence-manifest.md` 记录候选来源、脱敏检查、目标 RAW 路径和 SHA-256。
6. 提升后核对目标文件存在、hash 一致，且 WIKI `source` 指向该具体文件。

不提升：

- 导航过程和重复截图；
- 只用于定位控件的 DOM；
- 关闭后空页和无业务信息的失败尝试；
- 按钮清单、完成凭证和工具日志。

## 写入 WIKI

### Frontmatter

普通页面必须包含：

```yaml
---
title: 页面标题
type: module
source: |
  RAW_SOURCES/截图/示例系统/示例菜单/示例页面/YYYY-MM-DD_页面主体.png
date: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [示例系统, 示例页面]
related: [相关概念或流程]
---
```

普通页 `source` 必须指向 RAW 下存在的具体文件。不能伪造来源，也不能引用 `.work`。

### 页面正文

按实际内容裁剪：

```markdown
# 页面标题

## 页面定位
## 页面结构
## 筛选条件
## 列表字段
## 按钮与操作
## 弹窗 / 侧拉框 / 下拉
## 状态与提示
## 权限与风险
## 与其他页面、概念或流程的关系
## 信息边界
```

只写稳定事实：

```text
不推荐：本轮选择一条记录，点击修改，未点击确定。
推荐：“修改”打开普通弹窗；选择目标值后，“确定”会保存变更。
```

只有缺少样本、前置条件阻断或证据冲突会影响可信范围时，才保留一句简短信息边界。

### 文档颗粒度

- 普通弹窗、确认框、单步骤操作写在父页面。
- 完整新页面、复杂侧拉框、多步骤流程、独立审批或状态流转可独立成文。
- 多 Tab 页面由父 `_index.md` 记录稳定 Tab 列表和摘要；完成状态留 `.work`。
- 上级 Index 只展示真实菜单，不把页面内按钮伪装成同级菜单。
- Index 不写采集日期、完成打勾、受阻原因或恢复点。

### 冲突

页面、RAW、现有 WIKI、用户口径和源码冲突时：

1. 不直接覆盖。
2. 分开写正式业务规则、当前页面行为、源码实现和实现差异。
3. 在 `WIKI/待确认项/` 登记影响结论的决策问题。
4. 等确认后回写所有受影响页面。

采集进度不进入待确认项。

## Index 和日志

每个单元完成后更新：

- 最近 `_index.md`；
- 上层模块 index；
- `WIKI/index.md`；
- `WIKI/log.md`。

日志每次最多两条：

```markdown
## [YYYY-MM-DD] ingest|update | 标题
- 更新范围。
- 关键业务结论或遗留边界。
```

不在日志复制字段、按钮、截图路径、工具过程或完成凭证。

## 发布门

每个信息单元发布前：

1. RAW 只有最小充分证据。
2. WIKI 只有稳定知识和必要信息边界。
3. `source` 只指向永久 RAW。
4. Index 只表达稳定结构和摘要。
5. Log 不超过两条简短 bullet。
6. 运行：

```bash
python3 skills/lint/scripts/lint_wiki.py
python3 skills/record-feature-to-wiki/scripts/check_wiki_purity.py WIKI/本次修改文件.md
git diff --check
```

默认 lint 不通过时不能标记完成。Audit 提示单独报告：

```bash
python3 skills/lint/scripts/lint_wiki.py --audit
```

整批发布并验证后默认保留本批恢复点。只有用户要求清理，且清单中所有必要证据已提升、hash 已核对、WIKI source 有效、目标批次路径精确时，才清理该批目录；不得对 `.work/` 或知识库根执行宽泛递归删除。
