# Record Feature To Wiki

## 核心原则

把“看到的系统页面”变成“可追溯、可查询、可维护”的 WIKI。不要只写首屏结论；要验证页面结构、按钮真实形态、动态字段、宽表、弹窗、侧拉框、文件选择器，并把原始证据存入 `RAW_SOURCES/`。

## 前置硬规则

开始任何知识库维护前，必须先读仓库根目录 `AGENTS.md`，再读 `WIKI/index.md`、`WIKI/概念/概念_index.md` 和目标目录 `_index.md`。

## 单页面闭环

处理功能页面时，每次只处理一个页面、Tab、弹窗、侧拉框或配置项。当前信息单元必须完成以下闭环后，才能进入下一个：

1. 打开并确认页面。
2. 采集字段、按钮、提示、状态、必要截图。
3. 归档 RAW 证据。
4. 创建或更新对应 WIKI。
5. 更新必要 index/log。
6. 运行或计划运行 lint。

如果只是梳理菜单树，可以快速打开多个页面建立目录骨架；一旦开始记录字段、按钮、规则或流程，必须切回单页面闭环。

## 首次建库菜单采集

创建新知识库时，优先在用户监管下只读采集系统菜单结构。菜单结构应同步决定：

| 结构 | 落位规则 |
|------|----------|
| `RAW_SOURCES/截图/` | `系统/一级菜单/二级菜单/页面/截图文件` |
| `WIKI/功能模块/` | `系统/一级菜单/二级菜单/页面.md` |
| `WIKI/源码映射/` | 镜像功能模块路径，后续按页面或动作补代码证据 |

菜单采集阶段只记录菜单名、页面标题、URL 脱敏形式、页面层级和首屏截图；不要急着写完整规则。等骨架稳定后，再逐页补字段、按钮、弹窗、宽表、流程和源码证据。

## 必走流程

### 1. 预热

- 读 `AGENTS.md`。
- 读 `WIKI/index.md` 和 `WIKI/概念/概念_index.md`。
- 读目标目录 `_index.md` 和可能被更新的当前文件。
- 判断页面应落到哪个系统、菜单和 WIKI 路径。

### 2. 页面确认

记录：

| 信息 | 说明 |
|------|------|
| URL | 如含敏感参数，WIKI 中只写脱敏版本 |
| 页面标题 | 按页面可见文字记录 |
| 菜单路径 | 按系统真实菜单层级记录 |
| 当前环境/组织 | 如敏感，只用于操作确认，不写入可分享模板 |
| 角色/权限 | 只记录角色类别，不记录真实人员 |

### 3. 原始证据归档

- 截图放入：`RAW_SOURCES/截图/<系统>/<菜单>/<页面>/`。
- 下载模板放入：`RAW_SOURCES/设计文档/<系统>/<页面>/`。
- 专家补充放入：`RAW_SOURCES/专家知识/`。
- 用户纠正放入：`RAW_SOURCES/用户反馈/`。

### 4. 页面结构采集

必须检查：

- Tab、筛选区、更多条件。
- 按钮区、批量按钮、单行操作。
- 列表列头、固定列、横向滚动后的列。
- 分页、空状态、有数据状态。
- 弹窗、侧拉框、下拉联动、文件选择器。
- toast、二次确认、错误提示。

### 5. 操作权限

- 提交、保存、确定、删除确认等会改变数据的动作，点击前必须向用户申请权限。
- 取消、关闭、返回、展开、收起、查看下拉、打开弹窗但不提交，通常可直接执行。
- 对危险动作，优先停在二次确认层并记录提示。

### 6. 写 WIKI

- 使用标准 frontmatter：`title/type/source/date/updated/tags/related`。
- 普通页面必须写 `source`。
- 示例数据必须标注为示例，不得作为枚举全集。
- 跨模块有关联时，frontmatter `related` 和正文链接都要写。
- 表格多的文档优先整文件重写，避免破坏表格格式。

### 7. 同步索引和日志

- 更新最近目录 `_index.md`。
- 更新上层模块 index。
- 更新 `WIKI/index.md`。
- 追加 `WIKI/log.md`。

### 8. 验证

```bash
python3 skills/lint/scripts/lint_wiki.py
```

如 lint 发现既有问题，说明是否与本次变更有关。

## 采集检查清单

- [ ] 已读 `AGENTS.md` 和 index。
- [ ] 已确认页面标题、URL、菜单路径。
- [ ] 已采集首屏、更多条件、Tab、弹窗、侧拉框。
- [ ] 宽表已横向滚动确认完整列头。
- [ ] 按钮真实 UI 形态已确认。
- [ ] 落库动作未在授权前点击。
- [ ] RAW 已归档。
- [ ] WIKI 已更新。
- [ ] index/log 已同步。
- [ ] lint 已运行或已说明未运行原因。
