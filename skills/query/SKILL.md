# 查询业务知识库

先从 WIKI 的稳定业务正文回答，再按需追溯证据。不要把临时工作进度、历史日志、待确认问题或代码存在本身写成当前业务事实。

## 必读入口

依次读取：

1. `AGENTS.md`。
2. `WIKI/index.md`。
3. `WIKI/概念/概念_index.md`。

用户使用口语、旧称或截图化描述时，再读 `WIKI/概念/术语对照表.md`。

## 默认检索边界

默认只读：

- `WIKI/index.md`；
- 相关 `_index.md`；
- `WIKI/概念/`、`WIKI/功能模块/`、`WIKI/操作流程/` 的具体正文。

默认排除：

- `.work/`；
- `WIKI/log.md`；
- `RAW_SOURCES/`；
- `文档备份/`；
- 临时扫描表、恢复点和旧终审。

只有出现以下情况才定向突破默认边界：

| 需要 | 定向读取 |
|------|----------|
| 用户要求原始证据或质疑结论 | frontmatter `source` 指向的 RAW |
| 页面、状态或字段存在歧义 | 最相关的截图、模板或用户原话 |
| 用户要求隐藏校验、接口或算法依据 | 对应源码映射页及其版本 |
| 用户询问变更历史 | `WIKI/log.md` 和相关备份 |
| 当前结论受未决口径影响 | `WIKI/待确认项/` |

不得为了“多看一些”而遍历整个 RAW、`.work` 或备份。

## 查询流程

```text
理解问题
→ 读取全局入口
→ 沿 Index 选 2–5 个候选文档
→ 最多 3 轮局部检索
→ 综合回答并说明来源、版本和边界
```

### 1. 提取关键词

提取：

- 系统、角色、概念、页面、Tab、按钮、字段、状态；
- 口语、旧称、页面正式名称和源码字段名；
- 问题类型：概念、页面、流程、配置、源码、排障、历史或待确认；
- 输出目标：位置、步骤、规则、差异、字段、证据链或决策问题。

### 2. 按类型选择 Index

| 问题类型 | 优先入口 |
|----------|----------|
| 概念、术语、状态、规则口径 | `WIKI/概念/概念_index.md` |
| 页面、菜单、字段、按钮、Tab | `WIKI/功能模块/功能模块_index.md` |
| 跨页面操作、角色协作、排障 | `WIKI/操作流程/操作流程_index.md` |
| 隐藏校验、接口、算法、代码版本 | `WIKI/源码映射/源码映射_index.md` |
| 未决业务口径、页面待实采、实现差异 | `WIKI/待确认项/待确认项_index.md` |
| 不确定或跨模块 | 回到 `WIKI/index.md` |

Index 只用于定位，不把摘要或状态图标当成完整事实。

### 3. 迭代检索

最多 3 轮：

1. 使用 `rg` 在候选范围定位关键词。
2. 只读命中附近章节和最相关具体文档。
3. 判断是否足以回答。
4. 不足时扩展同义词、字段名、接口名或状态名。
5. 仍不足时停止扩展，说明知识缺口。

## 证据解释

### RAW

- RAW 是永久证据，不是默认业务查询层。
- 示例数据只证明样本存在，不能回答成枚举全集。
- RAW 与 WIKI 冲突时不直接覆盖结论；登记待确认项或切换维护流程。

### 源码映射

源码回答必须说明：

- 仓库、分支和 commit 是否明确；
- 工作区是否有未提交改动；
- 能力属于页面已提供、页面未提供、页面未实采、其他业务分支还是旧逻辑；
- 结论是否已经回写业务正文。

代码存在不等于当前页面可用，也不等于线上版本一致。

### 待确认项

- `待业务确认`、`待页面实采`、`待运行验证` 都不是已确认事实。
- `已确认待回写` 表示决策已经产生，但业务正文可能仍未同步；回答时必须说明这一过渡状态。
- 只有 `已关闭` 且相关 WIKI 已回写，才能把结论作为稳定知识直接回答。

### 旧终审

`.work` 中的 `final-audit.md` 只有在其基线 commit 或 WIKI tree hash 仍与当前状态一致时才有效。基线之后有业务文档变化，旧终审只能作为流程历史，不能证明当前工作树已经终审。

## 回答格式

```markdown
结论：...

依据：
- WIKI/功能模块/...
- WIKI/概念/...
- RAW_SOURCES/...（仅在查看后）
- WIKI/源码映射/...（仅在查看后）

边界：
- 当前页面尚未实采...
- 该实现基于本地源码 <commit>...
- 待确认项仍未关闭...
```

要求：

- 先给结论，再给依据。
- 清楚写出文件路径。
- 区分正式规则、页面行为、源码实现和实现差异。
- 不把 index 摘要、日志、恢复点、旧审计或示例值当成业务全集。

## 查询转维护

| 缺口 | 后续动作 |
|------|----------|
| 页面字段、按钮、弹窗或截图缺失 | `record-feature-to-wiki` |
| 隐藏校验、接口链路或算法规则缺失 | `extract-source-to-wiki` |
| 全仓、多仓或覆盖率任务 | `orchestrate-source-maintenance` |
| RAW 已存在但未结构化 | 按 `AGENTS.md` 的 RAW 提取模式 |
| 冲突需要业务决策 | 更新 `WIKI/待确认项/` |
| 断链、frontmatter、index、表格或纯度问题 | `lint` |

用户明确要求补齐时，切换对应维护流程并完成 WIKI/index/log/lint 闭环。
